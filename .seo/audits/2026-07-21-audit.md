## Executive Summary

Go Qatar has an **excellent engineering foundation** — 96 mobile / 100 desktop Lighthouse performance, perfect CLS, proper SSR/prerender, clean semantic HTML, and strong on-page titles/meta. The problems are almost entirely **SEO infrastructure and trust**, not code quality:

The site is effectively **invisible to search engines and AI answer engines** because the crawl/discovery layer is missing (no robots.txt, no sitemap, no llms.txt, no structured data), and it has a **duplicate-host problem** — apex `goqatar.app` and `www.goqatar.app` both serve 200 with no redirect, so **Google has canonicalized the site to `www.goqatar.app`** on its own while the 3 subpages remain **completely undiscovered** (0 clicks, 0 impressions over 28 days, confirmed via Search Console).

### Top 5 Critical / High Issues
1. **Host duplication + no canonical** — apex & www both 200, no redirect, no `<link rel=canonical>`; Google picked `www` for you. (Technical / Google API)
2. **No robots.txt & no sitemap.xml** (both 404) — no crawl guidance, no URL inventory; subpages never crawled. (Technical / Sitemap / Google API)
3. **Zero structured data** anywhere — no `SoftwareApplication`/`Organization`/`WebSite` schema on an app-marketing site (the single highest-value schema case). (Schema / Content / GEO)
4. **Missing security headers** — no CSP, X-Frame-Options, COOP, X-Content-Type-Options, Referrer-Policy, Permissions-Policy; HSTS lacks `includeSubDomains`/`preload`. (Technical)
5. **Weak E-E-A-T / trust** — no About/team, Gmail support address, template legal pages missing Qatar PDPPL & Apple-Sign-In disclosure, unsourced stats and decorative (fake) star ratings. (Content / SXO)

### Top 5 Quick Wins
1. Ship `robots.ts` + `sitemap.ts` (drafts already generated) and submit the sitemap in GSC.
2. Add a `metadataBase` + self-referencing canonical in `layout.tsx`; add a 308 redirect between apex ↔ www.
3. Add `Organization` + `WebSite` + `MobileApplication` JSON-LD (ready-to-paste in `findings/schema.md`).
4. Add security headers via `headers()` in `next.config.ts`.
5. Publish `llms.txt` (draft in `findings/geo.md`) and fix `/contact` `og:url` (currently points at homepage).

## Category Findings

### Technical SEO — 64/100
- **Critical** — apex & `www` both return 200 with identical ETags, no redirect → full duplicate-content exposure.
- **Critical** — no `<link rel="canonical">` anywhere; nothing consolidates the duplication or self-references pages.
- **High** — `robots.txt` 404 (no crawler directives, no sitemap pointer).
- **High** — `sitemap.xml` 404 (no machine-readable URL inventory).
- **High** — missing CSP, X-Frame-Options, COOP, X-Content-Type-Options, Referrer-Policy, Permissions-Policy headers (root cause: no `headers()` in `next.config.ts`).
- **Medium** — HSTS present but missing `includeSubDomains`/`preload`.
- **Low** — `/Privacy-Policy` case-variant 404s instead of redirecting; unlabeled `<select>`; footer heading-order skip.
- **Passes:** HTTP→HTTPS 308, trailing-slash 308, lowercase URLs, correct viewport, SSR prerender, index/follow, custom 404.

### Content Quality — 46/100 (weighted E-E-A-T ~33)
- **High** — no About/team/authorship; unclear who validates the Zone-Street-Building logic vs. Qatar's official addressing authority.
- **High** — personal Gmail support address (`help.goqatar@gmail.com`) instead of branded `@goqatar.app`; no legal entity/address despite soliciting B2B API leads.
- **High** — no JSON-LD; no canonical "What is Go Qatar" citable passage.
- **Medium** — thin, template legal pages; Privacy Policy omits Apple Sign-In, Google Maps SDK, analytics data flows, and any Qatar **PDPPL** reference.
- **Medium** — conflicting response-time claims (24h on Contact vs. 48h in Privacy/Terms).
- **Medium** — unverifiable enterprise "Business API" claims (SLA/white-label) with no docs/case studies.
- **Low** — decorative unsourced 5-star rating & "Loved by Qatar residents"; SVG mockups presented as "the real app"; stale "Last updated: June 2025".

### On-Page SEO — 68/100
- **Strengths:** strong title/meta/OG, clean H1→H2→H3, natural keyword coverage, Lighthouse SEO 100.
- **Critical** — no canonical tags (see Technical).
- **Medium** — `/contact` `og:url` incorrectly points to the homepage.
- **Low** — legacy `<meta name="keywords">` (harmless, no value); heading-order skip in footer.

### Schema / Structured Data — 28/100
- **High** — zero JSON-LD/Microdata/RDFa on any page.
- **High** — no `SoftwareApplication`/`MobileApplication` despite App Store + Play Store IDs being available.
- **Medium** — no `Organization`/`WebSite` entity graph (Go Qatar ↔ domain ↔ publisher Snap Infinity); no `BreadcrumbList`; no `ContactPoint`.
- **Info** — do NOT fabricate `aggregateRating` from decorative stars; skip deprecated `HowTo`; FAQ schema is AI-citation-only post May-2026 rich-result retirement.
- Ready-to-paste JSON-LD provided in `findings/schema.md`.

### Performance (CWV) — 90/100 (lab-only; no CrUX field data)
- Mobile: Perf 96, LCP 2.4s, CLS 0, TBT 50ms, SI 4.0s. Desktop: Perf 100, LCP 0.6s.
- **High** — LCP is render/hydration-driven (network finishes ~241ms); biggest lever.
- **High** — render-blocking CSS (~450ms potential savings via critical-CSS inline/defer).
- **Medium** — `431-*.js` 54% unused (~270ms); legacy-JS polyfills (~12KB); one forced reflow.
- **Note** — no CrUX field data (low traffic); re-validate against real-user data as traffic grows.

### AI Search Readiness (GEO) — 39/100
- **High** — no robots.txt (AI crawlers only accidentally allowed, no directives); no `llms.txt`; zero schema.
- **High** — no FAQ / no 134–167-word citable passages; copy is short marketing taglines with declarative headings.
- **High** — zero brand-mention signals (no YouTube/LinkedIn/Reddit/Wikipedia/social); bare Gmail contact.
- **Low** — rendering itself is strong (static prerender, full text without JS). `llms.txt` draft in `findings/geo.md`.

### Images — 70/100
- No oversized images (perf strong); OG image present (1200×630). 
- **Medium** — generic alt text ("Go Qatar", "Go Qatar icon"); illustrative SVG mockups instead of real app screenshots.

### Search Experience (SXO) — 33/100 (supplementary)
- **Critical** — page-type mismatch: SERP for "Qatar address finder / navigation app" is dominated by App/Play Store listings (~48%) and free government tools (QNAS, MOI "My Address", ~34%); a single marketing page can't compete without an in-browser lookup tool or store-listing rich results.

- **Medium** — no ownership of awareness-stage "what is Qatar's Zone-Street-Building system" content (won by ILoveQatar.net, Qatar Trendz).
