Executive Summary

Go Qatar has an excellent engineering foundation — 96 mobile / 100 desktop Lighthouse performance, perfect CLS, proper SSR/prerender, clean semantic HTML, and strong on-page titles/meta. The problems are almost entirely SEO infrastructure and trust, not code quality:

The site is effectively invisible to search engines and AI answer engines because the crawl/discovery layer is missing (no robots.txt, no sitemap, no llms.txt, no structured data), and it has a duplicate-host problem — apex goqatar.app and www.goqatar.app both serve 200 with no redirect, so Google has canonicalized the site to www.goqatar.app on its own while the 3 subpages remain completely undiscovered (0 clicks, 0 impressions over 28 days, confirmed via Search Console).

Top 5 Critical / High Issues

  1. Host duplication + no canonical — apex & www both 200, no redirect, no <link rel=canonical>; Google picked www for you. (Technical / Google API)
  2. No robots.txt & no sitemap.xml (both 404) — no crawl guidance, no URL inventory; subpages never crawled. (Technical / Sitemap / Google API)
  3. Zero structured data anywhere — no SoftwareApplication/Organization/WebSite schema on an app-marketing site (the single highest-value schema case). (Schema / Content / GEO)
  4. Missing security headers — no CSP, X-Frame-Options, COOP, X-Content-Type-Options, Referrer-Policy, Permissions-Policy; HSTS lacks includeSubDomains/preload. (Technical)
  5. Weak E-E-A-T / trust — no About/team, Gmail support address, template legal pages missing Qatar PDPPL & Apple-Sign-In disclosure, unsourced stats and decorative (fake) star ratings. (Content / SXO)

Top 5 Quick Wins

  1. Ship robots.ts + sitemap.ts (drafts already generated) and submit the sitemap in GSC.
  2. Add a metadataBase + self-referencing canonical in layout.tsx; add a 308 redirect between apex ↔ www.
  3. Add Organization + WebSite + MobileApplication JSON-LD (ready-to-paste in findings/schema.md).
  4. Add security headers via headers() in next.config.ts.
  5. Publish llms.txt (draft in findings/geo.md) and fix /contact og:url (currently points at homepage).
